Tech Stack :

The To-Do List Application is a task management tool built with Django. It features a robust backend with REST API and ORM, and a user-friendly frontend using HTML, CSS, and JavaScript. Manage tasks, authenticate users, reset passwords, and access FAQs with ease.

Full Stack Interaction :

The overall architecture includes the frontend (HTML/CSS/JavaScript) communicating with the Django API backend, which handles business logic and interacts with the PostgreSQL database.

  • Frontend: HTML/CSS/JavaScript interface for user interaction.
  • Backend: Django views/APIs interacting with DRF.
  • Database: PostgreSQL holding user data, tasks, FAQs, and authentication tokens.

Key Connections:

  • Frontend communicates with the API via AJAX or form submission.
  • Backend handles requests, applies business logic, and interacts with the database.
  • Data is persisted in the PostgreSQL database.

User Authentication API :

This API handles user authentication, including signing up, logging in, and password reset.

  • Frontend: Login form and signup form.
  • Backend: API endpoints for authentication and user management.
  • Database: User data, tokens, and profiles stored in PostgreSQL.

Key Connections:

  • User inputs credentials in UI.
  • Backend authenticates against user data in PostgreSQL.
  • Password reset tokens are generated and stored.

FAQ Management API :

The FAQ API provides a list of frequently asked questions. The backend serves the data from the FAQ model using Django ORM and DRF.

  • Frontend: User interface to display FAQs.
  • Backend: Django views serve FAQ data.
  • Database: FAQ data storage and retrieval.

Key Connections:

  • Frontend requests FAQ data.
  • Backend responds with FAQ data.
  • Frontend displays the FAQ list.

Task Management API :

The task management API handles CRUD operations for tasks in the to-do list.

  • Frontend: Form submission to create or update tasks.
  • Backend: Django API views handling POST, PUT, DELETE requests.
  • Database: Tables like Task, User, and associations.

Key Connections:

  • User inputs tasks via UI.
  • Backend API processes input and interacts with PostgreSQL.
  • Frontend renders updated task list.
